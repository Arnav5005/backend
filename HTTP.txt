HTTP stands for Hyper Text Transfer Protocol

It is the foundational, application-layer protocol used for transmitting data—such as HTML, images, and videos—across the World Wide Web. It operates on a client-server model, where a browser (client) sends a request to a server, and the server returns the requested data

URL - Uniform Resource Locator 
    Purpose - Locate a resource (address)
    Includes location - Yes
    ex - https://example.com

URN - Uniform Resource Name
    Purpose - Name a resource (persistent)
    Includes location - No
    ex - urn:isbn:0451450523

URI - Uniform Resource Identifier (https://example.com)
    Purpose - Identify a resource
    Includes location - Sometimes
    ex - https://example.com

HTTP methods :

Basic set of operations that can be used to interact with server

. GET : retrieve a resource
. HEAD : No message body (response headers only)
. OPTIONS : what operations are available 
. TRACE : loopback test (get some data)
· DELETE : remove a resource
· PUT : replace a resource
. POST : interact with resource (mostly add)
· PATCH : Change part of a resource

HTTP status code

. 1XX : Informational 
· 2XX : Success
· 3XX : Redirection
. 4XX : Client error
· 5XX : Server error
